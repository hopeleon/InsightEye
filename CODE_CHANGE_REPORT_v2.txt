================================================================================
 InsightEye 代码改动详细对比报告
================================================================================

================================================================================
 文档概述
================================================================================

本文档记录了将 InsightEye 项目从阿里云 DashScope 云端 ASR 改造为本地
FunASR + CAM++ 声纹识别的代码改动。

--------------------------------------------------------------------------------
 改动动机
--------------------------------------------------------------------------------

 问题               原版方案                          本地化方案
--------------------------------------------------------------------------------
 成本               按调用计费                        一次性模型，零成本
 隐私               音频上传阿里云                    音频完全保留本地
 网络               必须联网                          完全离线可用
 延迟               依赖网络传输                      本地推理，更低延迟
 声纹               无，仅靠文本推断                  CAM++ 精准识别

================================================================================
 二、核心文件详细对比
================================================================================

--------------------------------------------------------------------------------
 2.1 app/config.py - 配置文件
--------------------------------------------------------------------------------

 【改动要点】
 - 移除：DASHSCOPE_API_KEY、DASHSCOPE_REALTIME_WS_URL 等云端配置
 - 新增：LOCAL_MODEL_DIR、FUNASR_MODEL_DIR、CAMPPLUS_MODEL_DIR 等本地配置
 - 新增：LOCAL_DEVICE 配置（cuda/cpu）

--------------------------------------------------------------------------------
 2.2 app/realtime_ws_server.py - WebSocket 服务（改动最大）
--------------------------------------------------------------------------------

 【原版架构】
 浏览器 → WebSocket → 本地桥接服务 → 阿里云 DashScope → ASR返回

 【本地化架构】
 浏览器 → WebSocket → 本地流式管道
                            ↓
                  ┌─────────────────────┐
                  │ StreamingPipeline    │
                  │  ├─ StreamingVAD     │  语音活动检测
                  │  ├─ StreamingASR     │  语音转文字
                  │  └─ CAM++声纹识别   │  说话人比对
                  └─────────────────────┘

 【新增功能】
 - 自动声纹注册（第一人→面试官，第二人→候选人）
 - 手动声纹注册（start_registration 消息类型）
 - 声纹相似度计算（阈值 0.75）

--------------------------------------------------------------------------------
 2.3 app/realtime_session.py - 会话状态管理
--------------------------------------------------------------------------------

 【新增字段】
 - speaker_recognizer：CAM++ 说话人识别器
 - voice_registered：声纹是否已注册
 - voice_mapping：声纹映射 {"interviewer": "speaker_a", ...}
 - sequential_roles：发言顺序角色推断

 【新增方法】
 - update_voice_mapping()：更新声纹映射
 - set_speaker_recognizer()：设置说话人识别器

 【Segment 数据结构新增字段】
 - recognized_role：声纹识别的角色
 - speaker_confidence：最佳相似度
 - interviewer_sim / candidate_sim：与各角色的相似度

--------------------------------------------------------------------------------
 2.4 app/realtime_ws_state.py - 事件状态处理
--------------------------------------------------------------------------------

 【新增函数】
 - consume_local_transcript_event()：处理本地 FunASR 转录完成事件

 【角色推断逻辑变化】
 - 原版：文本特征分析（role_inference.py）
 - 本地化：声纹比对 + 发言顺序 + 回溯修正

--------------------------------------------------------------------------------
 2.5 app/realtime_analyzer.py - 实时分析器
--------------------------------------------------------------------------------

 【build_realtime_transcript 函数变化】
 - 参数新增：voice_mapping、sequential_roles
 - 角色分配优先级：recognized_role > voice_mapping > sequential_roles

--------------------------------------------------------------------------------
 2.6 app/role_inference.py - 已移除
--------------------------------------------------------------------------------

 功能已整合到 realtime_ws_state.py 和 realtime_analyzer.py 中


================================================================================
 三、新增文件详解
================================================================================

--------------------------------------------------------------------------------
 3.1 app/model_manager.py - 模型管理器
--------------------------------------------------------------------------------

 【核心功能】单例模式，统一管理所有本地模型
 - FunASR 模型（VAD + ASR）
 - CAM++ 中文声纹模型
 - CAM++ 英文声纹模型
 - Silero VAD 模型

 【模型加载】在后台线程中加载，避免阻塞主线程

--------------------------------------------------------------------------------
 3.2 app/streaming_pipeline.py - 流式处理管道
--------------------------------------------------------------------------------

 【核心类】
 - StreamingVAD：流式语音活动检测，检测语音边界和说话人切换
 - StreamingASR：流式语音识别，使用 FunASR
 - StreamingSpeakerRecognition：流式声纹识别，实时比对说话人
 - StreamingPipeline：完整管道，协调以上组件

 【特性】
 - 实时流式处理，无需等待完整音频
 - 说话人变化检测
 - 声纹识别异步比对

--------------------------------------------------------------------------------
 3.3 app/speaker_recognition.py - 说话人识别模块
--------------------------------------------------------------------------------

 【核心类】
 - SpeakerRecognizer：说话人注册和识别
 - SpeakerEmbeddingExtractor：声纹特征提取（CAM++）
 - InterviewSpeakerManager：面试场景的说话人管理

 【功能】
 - 声纹注册：register_speaker()、register_embedding()
 - 声纹识别：identify_speaker()
 - 增量更新：update_speaker_embedding()

--------------------------------------------------------------------------------
 3.4 app/vad_asr_pipeline.py - VAD + ASR 异步管道
--------------------------------------------------------------------------------

 备选实现，采用异步架构：
 - VADProcessor：Silero VAD 语音活动检测
 - VADASRPipeline：异步处理循环，队列+线程池

--------------------------------------------------------------------------------
 3.5 local_settings.py - 本地配置文件
--------------------------------------------------------------------------------

 【配置项变化】
 - 移除：DASHSCOPE_API_KEY 等云端配置
 - 新增：USE_LOCAL_ASR、LOCAL_DEVICE 等本地配置


================================================================================
 四、架构变化总结
================================================================================

--------------------------------------------------------------------------------
 4.1 整体架构对比
--------------------------------------------------------------------------------

 【原版架构（阿里云）】
 浏览器 → WebSocket → 桥接服务 ─(WSS)→ 阿里云DashScope → ASR返回

 【本地化架构】
 浏览器 → WebSocket → 本地流式管道
                            ↓
                    ┌───────────────┐
                    │ Silero VAD    │ 语音检测
                    │ FunASR ASR    │ 转文字
                    │ CAM++ 声纹    │ 说话人识别
                    └───────────────┘

--------------------------------------------------------------------------------
 4.2 数据流对比
--------------------------------------------------------------------------------

 【原版】
 音频 → 桥接服务 → 阿里云 → 转录文本 → 角色推断 → 会话存储

 【本地化】
 音频 → VAD检测 → 声纹识别 → 转录文本 → 角色分配 → 会话存储
                     ↓
               自动声纹注册
               第一人→面试官 / 第二人→候选人

--------------------------------------------------------------------------------
 4.3 角色识别流程对比
--------------------------------------------------------------------------------

 【原版】收集片段 → 文本特征分析 → 推断角色
 【本地化】声纹已注册→直接比对 / 未注册→发言顺序推断 + 回溯修正

 六、迁移指南
================================================================================

--------------------------------------------------------------------------------
 6.1 迁移步骤
--------------------------------------------------------------------------------

 1. 安装依赖：pip install funasr torch numpy addict pyyaml
 2. 下载模型到 models/ 目录
 3. 更新配置：USE_LOCAL_ASR = True
 4. 移除 DashScope 相关配置

--------------------------------------------------------------------------------
 6.2 硬件要求
--------------------------------------------------------------------------------

 GPU：NVIDIA 4GB+（推荐 8GB+）
 内存：8GB+（推荐 16GB+）
 存储：5GB+（推荐 10GB+）

--------------------------------------------------------------------------------
 6.3 性能对比
--------------------------------------------------------------------------------

                  原版(阿里云)     本地化版
--------------------------------------------------------------------------------
 ASR延迟          200-500ms       100-300ms
 首包延迟         依赖网络         更低
 成本             按调用计费       一次性成本
 离线能力         否              是


================================================================================
 七、总结
================================================================================

本次改动将 InsightEye 从云服务依赖的演示系统转变为完全本地化部署的生产级系统：

1. ASR引擎：阿里云 DashScope → FunASR
2. VAD检测：阿里云 server_vad → Silero VAD
3. 声纹识别：无 → CAM++ 本地模型
4. 架构：桥接服务 → 本地流式处理

改动后系统具有：零成本、完全隐私、离线可用、更低延迟、精准说话人识别
